Enumerable group_by
10/12/2014

group_by is a ruby method that was introduced in version 1.9. The group_me method interates through the array, checking each element to the block of code. Take this code for example: (0..30).group_By {|n| n % 3}. The numbers 0 through 30 are first passed through the block then they are assigned an integer. Then they are grouped by the result value of the block of code. In this case, they would grouped by the remainder they receive, starting from the numbers that have no remainder and are divided perfectly by 3. You can get pretty creative with how you approach the use of group_me, for example you can use the group_me method to group the arrays by the length of the string.

Use Arrays and Hashes the Right Way
10/04/2014

Hashes are a lot more flexible between the two. When it comes to speed in finding a specific object, the search with a hash is a lot faster. Programmers will use hashes when they need to map a value with a specific name or string, hence the flexibility. Although, you have to keep in mind that hashes aren't distributed as ordered data, but they are distributed as insertion order. The best times to use hashes are when you need to name parameters in a function. It just creates a cleaner look for your code.

Arrays are mostly just ordered objects that are indexed by integers, which with hashes you can index it by any string. When you define a method with multiple arguments and objects, they often return the data as an Array. When trying to reference an object in an array, the positions are stored as integers starting from 0. For example let's say you have an array with multiple values: football_teams['broncos', 'chargers', 'chiefs, 'raiders'] If you wanted to reference to 'broncos' you would type football_teams[0]. For the 'Chiefs' you would type football_teams[2].

Layers Upon Layers. Understanding Margin, Border, and Padding
9/27/2014

To any developer's eye, every element in html are seen as boxes. Hence the picture you see below, which is known as the "box model". The box model is a good thing to know when you are positioning or making changes to the actual content. Starting from the center, the content is the part of the element in which the images and text appears. The padding is the clear area in between the content and the border. The border is the area that goes around the padding and content which keeps everything in a box. And finally, the margin is the clear area outside the border which can almost be seen as the "padding" outside everything else.

A common question asked is when is the appropriate time to use margin instead of padding or vice versa. One thing you have to understand is that padding is actually considered to be part of the element. Which means that padding is NOT prone to collapsing when other elements are trying to do so. Margins a little different. Since margins are the clear areas outside the element itself, they don't neccessarily act as solidified content. For example if two elements were sitting next to each other with the same pixels of margin, lets say 5px. The distances between these two elements will only be 5px and not a combined 10px. Why? Because the margins just overlap, they don't bump each other around.

Git and Version Control - (Insert cheesy "git" pun here)
9/17/2014

"Code doesn’t exist unless it’s checked into a version control system. Use version control for everything you do. Any version control, SVN, Git, even CVS, master it and use it." - Jeff Atwood

I don't see how anyone serious about programming could turn away from using version control. Version control is so benefical to developers because it allows you to create both an archive and a simple approach to undo corrupted code. Not to mention that its extremely useful when working in large groups because project members can make changes to the same repository and have it merged after it commits. Version control opens up whole new opportunities to experiment a little more with projects without running into any repercussions. We all know that feeling way too well when you are deeply invested into a project, almost at the finish line but you run into a problem that is irreparable at that point. Version control allows you to undo adverse changes and create a smoother workflow.

brb, installing Git for real life.

Out of all the wonderful VCS(version control systems) applications out there, Git is the most prominent one. Git helps me keep track of changes in my code because it retains all the history of my files. This gives me the ability to see the changes I've made over time and roll back any changes I don't find appropriate. Git has built in commands in which you can check the log of all the changes made to a specific repositories. Since Git uses more than just one repository, it's utilized a lot more in all kinds of ways. Git allows you to have your own privary repository in which you can make changes to without having to merge your work to the central repository.

GitHub currently is the trendiest web-based code hosting server out there. It makes sense to use Github to store your code because it has the largest userbase, which means there are a lot more opensource projects available to you. When making any big decisions between two products that you like, you want to always go with the one with the better track record. I expect myself to stick to GitHub for as long I am a software developer and don't plan on making any changes any time soon.
