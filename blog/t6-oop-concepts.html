Classes vs. Modules
10/26/2014

In ruby, everything is an object. All objects need functions and behaviors to do things, which is why we have modules. Classes as you know allows us to create a set of behaviors(methods) for the explicit receiver. The superclass of class are modules and the superclass of modules are objects. Unlike classes, modules cannot be intantiated.

To understand modules, you must first understand how they are used in the world of Ruby. Modules are used to aggregate a bunch of instance methods and used to define a name-space. Since modules have the ability to have multiple classes, it is a useful way to have multiple inheritrance, since ruby only allows single inheritance.
