The Correlation Between Classes and Real Life
10/19/2014

Classes and data structures can model real life objects quite substantially. Since Ruby is an Object Oriented programming language, it uses classes to blueprint objects and its behavior. The data structures are pretty much the 'characteristics' of a certain object. Now, before I start spitting a bunch of nonsense words at you, lets make a comparison with a real life object. Take a fishing reel for example. A fishing reel consists of a turning handle, a spool for fishing line, and drag control to set the resistance of it. These characteristics are the data stuctures. Since, not all fishing reels have the same spool size, drag power and gear ratios, these unique characteristics and data input makes this particular fishing reel genuine to the class FishingReels.

A fishing reel has certain functions such as turning the handle will allow it to retrieve line, setting it to free-spool will allow it cast, and setting the drag will increase or lower resistence depending on how big the fish is. These are the functions of a fishing reel which are the methods of the class FishingReel.
